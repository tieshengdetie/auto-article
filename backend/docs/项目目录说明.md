# 项目目录说明

本文档详细描述了 `auto-article` 后端项目的目录结构和各模块职责。

## 项目整体结构

```
backend/
├── api/                    # API 控制层
├── command/                # 命令行工具（Cobra）
├── common/                 # 公共组件
├── config/                 # 配置管理
├── constants/              # 常量定义
├── docker/                 # Docker 部署配置
├── docs/                   # 文档
├── global/                 # 全局变量
├── initialize/             # 初始化模块
├── logs/                   # 日志目录
├── middleware/             # 中间件
├── prompt/                 # 提示词管理
├── router/                 # 路由配置
├── service/                # 业务逻辑层
├── utils/                  # 工具函数
├── go.mod                  # Go 依赖管理
├── main.go                 # 项目入口
└── application.*.yml       # 环境配置文件
```

## 目录详解

### 1. api/ - API 控制层

负责处理 HTTP 请求，参数校验，调用 Service 层，并返回响应。

| 子目录 | 说明 |
|--------|------|
| `hotSearch/` | 热搜相关接口 |
| `news/` | 新闻相关接口 |

每个 API 模块包含：
- `dto/` - 请求参数定义（Data Transfer Object）
- `vo/` - 响应结果定义（View Object）
- `xxx.go` - 控制器实现

### 2. command/ - 命令行工具

基于 Cobra 框架的命令行工具。

| 文件 | 说明 |
|------|------|
| `cmd/root.go` | 根命令定义 |
| `cmd/server.go` | 启动服务命令 |
| `main.go` | 命令行入口 |

项目共用的核心组件。

|--------|------|

## 核心架构

### 分层架构

```
┌─────────────────────────────────────────────────────────────┐
│                      HTTP 请求                              │
└───────────────────────────┬─────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                      router/                               │
│                    路由分发                                 │
└───────────────────────────┬─────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                      api/                                  │
│                  控制层（参数校验、响应封装）                 │
└───────────────────────────┬─────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                      service/                              │
│                   业务逻辑层                                │
└───────────────────────────┬─────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│          common/          │           utils/               │
│  公共组件（LLM、Kafka等）  │      工具函数                  │
└─────────────────────────────────────────────────────────────┘
```

### 数据流向

1. **请求进入** → Router 分发 → API 层校验 → Service 层处理 → 返回响应
2. **Service 层**可调用：
   - `common/llm/` - 大模型接口
   - `common/sendHttp/` - 第三方 HTTP 接口
   - `utils/` - 工具函数

## 配置文件

项目支持多环境配置：

| 文件 | 环境 |
|------|------|
| `application.dev.yml` | 开发环境 |
| `application.test.yml` | 测试环境 |
| `application.prod.yml` | 生产环境 |

通过命令行参数 `-envString` 指定环境：
```bash
go run main.go -envString dev
```

## 启动方式

```bash
# 开发环境
make runDev

# 生产环境
make runProd

# 构建
make build
```

## 技术栈

- **语言**: Go 1.22
- **框架**: Gin
- **数据库**: MySQL + GORM
- **缓存**: Redis
- **消息队列**: Kafka、RabbitMQ
- **日志**: Zap
- **大模型**: 豆包、阿里云千问
- **命令行**: Cobra
